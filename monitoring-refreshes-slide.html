Monitoring Foreign-Catalog Metadata Refreshes

See every refresh — and catch the out-of-memory ones — from one place. No per-table digging.

1 Start here — one query
system.query.historySQL
SELECT start_time, statement_text, execution_status,
       total_duration_ms, error_message
FROM   system.query.history
WHERE  statement_text ILIKE 'REFRESH FOREIGN TABLE%'
ORDER BY start_time DESC;
How to read the results
FINISHEDRefresh worked.
FAILED · OOMOut of memory — TASK_FAILED_EXECUTOR_LOSS / oom / GC overhead. Retry on a bigger cluster.
FAILED · OTHERTABLE_OR_VIEW_NOT_FOUND / permission → name or access issue (not memory).

Long total_duration_ms ≈ full refresh  ·  short ≈ incremental
